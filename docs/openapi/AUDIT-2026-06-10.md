# Midaz OpenAPI / Swagger Documentation Audit

_Date: 2026-06-10 · Branch: feat/monorepo-consolidation · Method: 6 parallel finder agents + synthesis (workflow `openapi-doc-audit`)_

_Scope: `components/ledger` (native onboarding+transaction, folded-in CRM + Fees), `components/tracer`, `components/reporter`, shared schemas (`pkg/mmodel`, `feeshared/model`, `pkg/reporter/model`, tracer `pkg/model`), and the Postman hub generation pipeline._

## 1. Verdict

Midaz has **one excellent reference surface and a long tail that fails to meet it** — and the gap is not where the brief guessed. `pkg/mmodel` (shared ledger domain) and tracer's **endpoint-level** annotations are genuinely strong; the real damage is concentrated in three places:

1. A **security model that doesn't work as documented** — ledger declares `BearerAuth` but zero operations reference it via `@Security`, so the spec advertises an auth scheme that applies to nothing.
2. **Schema-layer rot outside `mmodel`** — tracer exposes raw domain entities with zero swag annotations, the fee billing family is persistence-shaped bson structs on a public surface, and reporter documents unexported Go types as response contracts, producing ugly `github_com_LerianStudio_...` names across all three specs.
3. A **doc pipeline broken by the reporter consolidation** — `generate-docs.sh` still targets the dead `reporter-manager` binary, so a clean `make generate-docs` cannot reproduce the committed reporter spec, and the published hub copy has already drifted (missing the `Partial` enum).

Cross-component parity is poor on nearly every general-info field (version format, contact, license, schemes, title branding), with no source of truth enforcing it. The honest framing: a **mature, well-instrumented codebase with documentation discipline that decayed at the edges during consolidation** — not a greenfield mess. Fixes are mostly mechanical and high-leverage.

## 2. Findings by severity

### CRITICAL

**C1 — `BearerAuth` security scheme is declared but no ledger operation references it (`@Security` absent everywhere)**
- Component: **ledger** (native + folded-in surfaces) · Dimension: annotation-quality / security
- Evidence: `components/ledger/cmd/app/main.go:30-33` declares `@securityDefinitions.apikey BearerAuth`; zero `@Security` hits across all native handlers (e.g. `organization.go:38`, `transaction.go:44`); auth is instead modeled ad-hoc as an optional `@Param Authorization header string false` on every endpoint.
- Why critical: the generated spec defines a security scheme that **no operation requires** (dangling), and Swagger UI's Authorize button binds to `BearerAuth` and applies to nothing. A consumer can't tell endpoints are authenticated; a generated SDK won't attach the token. Worst of both worlds — dangling definition **plus** duplicated header params.
- Recommendation: Add `@Security BearerAuth` to authenticated operations and drop the per-endpoint `Authorization` `@Param`. The scheme exists, the auth is real (`ProtectedRouteChain`), so document it as a requirement. **tracer and reporter already use `@Security`; this defect is ledger-only.**

### HIGH

**H1 — Tracer exposes raw domain entities on the wire with zero swag annotations**
- tracer · schema-organization · `components/tracer/pkg/model/rule.go:43-62` (0 `swagger:model`, 0 `@name`, 0 field descriptions); `rule_validation.go:307-311` returns `model.Rule` directly. Generated `Rule` renders as `github_com_LerianStudio_midaz_v4_components_tracer_pkg_model.Rule`, all fields untyped `string`. Same for `Limit`, `Scope`, `AuditEvent`, `TransactionValidation`, `ValidationResponse`. 15 example tags across 177 fields, 0 field descriptions.
- Fix: response DTOs in tracer `http/in` mirroring the ledger pattern, or at minimum `@name` + per-field `example`/`description`/`enums`. Prioritize `Rule.Expression` (CEL string), the `Decision`/`RuleStatus`/`LimitStatus`/`LimitType` enums, `AuditEvent.EventType`.

**H2 — Fee billing schema family has no swag annotations and is persistence-shaped (bson) on a public surface**
- ledger (folded-in fees) · schema-organization · `components/ledger/pkg/feeshared/model/billing_package.go:44-62,109-128` — 7 wire structs (`PricingTier`, `DiscountTier`, `AccountTarget`, `EventFilter`, `BillingPackage`, `BillingPackageUpdate`) with no `swagger:model`/`@name`/`@Description`, only json+**bson** tags. 12 leaked `feeshared_model.*` definitions in `components/ledger/api/swagger.json`. Same package's `fees.go:24-82` **is** fully annotated — bar is reachable, just not applied.
- Fix: raise `billing_package.go` to the `fees.go` bar. bson tags on a wire model is a smell — add a thin response DTO if both persistence and wire.

**H3 — Folded-in (CRM + Fees) endpoints omit response-description strings on every `@Success`/`@Failure`**
- ledger (all 25 CRM + fee endpoints) · annotation-quality · `holder.go:35-38`, `fees_package_handler.go:54-60` (bare `@Success 201 {object} mmodel.Holder`) vs native `accounttype.go:35-40` (`... "Successfully created account type"`). Generated OpenAPI loses per-status semantics for the entire folded-in surface.
- Fix: description strings on every `@Success`/`@Failure` across the 8 folded-in handler files; prioritize 409/422/404.

**H4 — CRM handlers omit `401`/`403` (and `409` on create) that fee and native handlers declare**
- ledger (Holders, Instruments, Composition, holder-accounts) · annotation-quality · `holder.go:36-38`, `instrument.go:39-41` declare only `400/404/500` despite `ProtectedRouteChain`; fee siblings + native `accounttype.go` declare `401/403/409`. Two error vocabularies in one binary; CRM half strictly thinner.
- Fix: add `401/403` to authenticated CRM endpoints, `409` to CreateHolder/CreateInstrument, `422` where business validation fires.

**H5 — Doc generator targets the dead `reporter-manager` component; a clean run cannot reproduce the reporter spec**
- reporter / pipeline · `postman/generator/generate-docs.sh:20,88,97` — `COMPONENTS=("ledger" "tracer" "reporter-manager")` runs `swag init` against `components/reporter-manager` (Dockerfile-only anchor, no `cmd/app/main.go`). Real binary is `components/reporter` (commit `bfa9b4b69`). `make generate-docs` fails at the reporter step on a clean checkout, yet committed `components/reporter/api/*` exist — produced out-of-band.
- Fix: `COMPONENTS=("ledger" "tracer" "reporter")`, regenerate, confirm identical. Also fixes H6's publish destination.

**H6 — Published hub copy lives under stale `postman/specs/reporter-manager/` and has drifted from live source**
- reporter / pipeline · folder-organization · `generate-docs.sh:148` (dead name becomes published dir). `postman/specs/reporter-manager/swagger.json:1473` lacks the `Partial` enum that live `components/reporter/api/swagger.json:1473` has — content drift, not just naming.
- Fix: after H5, `git mv postman/specs/reporter-manager postman/specs/reporter`; add a CI check diffing `postman/specs/<c>` against `components/<c>/api`.

## 4. Schema organization assessment

Schema quality is **sharply tiered, and the tiers correlate with distance from `pkg/mmodel`.** `pkg/mmodel` is the gold standard — disciplined `@Description`, examples, `format`/`enum`/`maxLength` hints, clean `@name`. That discipline does not survive the move outward. Tracer's `pkg/model` is the floor: zero swag annotations, raw domain entities served directly as the wire contract. The fee billing family (`feeshared/model/billing_package.go`) is bson-tagged persistence structs masquerading as wire models — and the same package's `fees.go` is fully annotated, so it's neglect, not a structural constraint. Reporter sits between: clean input models, but response entities named by Go import path, unexported types as public contracts, untyped `Pagination.items`, and a critical triple-nested `Filters` map whose operator semantics live only in Go comments.

Two structural smells worth naming as policy: **(1) bson tags on a struct that doubles as an OpenAPI model** (feeshared billing) — persistence leaking onto the wire, the same anti-pattern the project's own streaming-events rule forbids; and **(2) `@name` omission**, the single cheapest, highest-volume fix — a mechanical sweep that removes the ugly `github_com_LerianStudio_...` names across all three specs at once.

## 5. Recommended action order

### Quick wins (mechanical, low blast radius)
1. **Fix the doc generator (H5) + re-sync the hub (H6).** Rename `COMPONENTS` `reporter-manager`→`reporter`, regenerate, `git mv` the stale dir, add a CI drift check. Only item where committed artifacts can't currently be reproduced — fix first.
2. **`@name` sweep across all wire structs (M13/L12).** Start `mmodel.Balance` + `mmodel.*Settings`, then feeshared, reporter response entities, then tracer. Kills 18+ leaked names in ledger and nearly all of tracer's in one pass.
3. **Fix wrong/contradicting examples and copy-paste defects (M2, M11, L2, L3, L5, L6).**
4. **General-info parity pass (M4, M5, M7, L14, L15, L16, L17).** Copy ledger's `@contact`/`@license` into tracer+reporter, normalize `@version`/`@schemes`/`@title`, fix the swagger.io scaffold URL, add a parity checklist + CI assertion.
5. **Normalize `@Tags` (M6) + `@Router` method case (L1).** Title-Case-plural repo-wide; lowercase methods; grep guards.

### Larger efforts (require judgment or new DTOs)
6. **Resolve ledger security model (C1) — highest severity.** Add `@Security BearerAuth` to authenticated ops, drop the per-endpoint `Authorization` `@Param`.
7. **Annotate tracer's schema layer (H1) + the fee billing family (H2).** Biggest schema-quality gain; prioritize `Rule.Expression` (CEL) and enum fields.
8. **Lift ledger's thin tier + folded-in surface to the native bar (M1, H3, H4, M14, L4).** Response-description strings, consistent 401/403/409/422/404, UUID-format hints, behavioral params.
9. **Make the reporter `Filters` body discoverable (M10) + add missing `422`/`404` (M3).**
10. **Resolve related-party surface (M9) + fee-estimate type leak (L8).**
